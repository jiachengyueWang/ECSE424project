Design Evolution

After developing the alpha system, we collected the feedback the evaluating team gave us. Combining the feedback from the peer and the aspects that our team wants to modify. We came up with 3 evolutions below.


  • 1. Language selection tool is added:

    • To better achieve the idea of getting more users involved in a more customized environment. In this Beta system, we have implemented the language selection functionality into EasyAsk. It supports three languages at the moment, English, French and Mandarin. With this being done, we believe we will explore a wider user market and try to find opportunities to get feedback after their user tests in order to get improvements for our products.

  • 2. Collecting user feedback page:

    • The modification to our prototype is for the convenient communication environment for both users and the design team. In the Beta system, our design will actively focus on the user feedback to better improve and complete our design and system. Specifically, we provided a pop-up window to collect the user experience with our application.

    The feedback page includes a quick selection between good experience (indicated by a check mark) and a bad experience (indicated by a cross), it also includes an email address so that if the user has more to say, he / she can email the suggestions by contacting us with the address.

  • 3. Clearer user interface is now being modified :

    • There is no actual connection of the prototypes, so we added notes to explain in the alpha system. However, It was brought up one one of our test subjects that the notification of explaining presenter mode and participant mode is not clear since it is in the middle of the function and it is unaesthetic.

    Because a lot of confusion is created in the testing, we tried to change the font, size, color and position of the message for a clearer view from the user side. In this prototype, the notification message appears gently with suitable choices of font and color. Now, users can get the message from our system and continue operating our design..


Prototype Revisions

We modified our design of the Beta System to improve on the Alpha System. By focusing on feedback and being realistic, we minimized unnecessary interactions and reduced on-click feedback time for the system. We decided to reevaluate the feasibility and development effort necessary to make the EASYASK tool functional as an inbuilt tool in Zoom. We have added a disclaimer that the users in the beta system are not yet connected in the real-life Zoom app because Zoom system is not open source
